Documentación del Lenguaje C3D

Referencia completa del lenguaje de Código de Tres Direcciones implementado en este playground educativo.

1. Introducción

El Código de Tres Direcciones (Three-Address Code o TAC) es una representación intermedia utilizada en compiladores para facilitar el análisis, optimización y generación de código. Se caracteriza por ser un lenguaje de bajo nivel que descompone expresiones complejas en instrucciones simples, donde cada instrucción realiza, como máximo, una operación básica.

Esta implementación es una versión simplificada con fines educativos, diseñada para ilustrar los conceptos fundamentales de una Representación Intermedia (IR) en el contexto de la construcción de compiladores. El lenguaje implementado modela:

  • Asignaciones simples y con operadores binarios
  • Operaciones aritméticas y relacionales
  • Control de flujo mediante etiquetas y saltos
  • Operaciones de entrada/salida básicas

El propósito de este playground es proporcionar un entorno interactivo para comprender cómo funcionan las representaciones intermedias, cómo se estructuran los bloques básicos y cómo se construye el grafo de flujo de control de un programa.

2. Variables

Las variables en C3D son dinámicas y se crean implícitamente en el momento de su primera asignación. No requieren declaración previa ni especificación de tipo.

Características

  • Nombres: Identificadores alfanuméricos que comienzan con letra o guion bajo (ej. t1, contador, x)
  • Creación implícita: Se crean automáticamente al realizar la primera asignación
  • Tipado dinámico: Todas las variables almacenan valores numéricos (punto flotante)
  • Alcance global: Todas las variables son accesibles en todo el programa

Literales Numéricos

El lenguaje soporta literales numéricos enteros y de punto flotante:

  • Enteros: 5, 42, -10
  • Punto flotante: 3.14, -2.5, 0.001

Ejemplos

t1 = 5
x = t1
contador = 0
pi = 3.14159
temp = x

Nota: No existe distinción entre variables temporales (como t1) y variables con nombres descriptivos. Todas se tratan de la misma manera por el intérprete.

3. Expresiones Aritméticas

Las expresiones aritméticas en C3D siguen el formato de tres direcciones, donde cada instrucción contiene como máximo un operador binario.

Operadores Soportados

Operador Descripción Ejemplo Resultado
+ Suma t1 = a + b Suma de a y b
- Resta t2 = a - b Resta de a menos b
* Multiplicación t3 = a * b Producto de a y b
/ División t4 = a / b División de a entre b

Forma General

destino = operando1 operador operando2

Donde:

  • destino: Variable que recibirá el resultado
  • operando1, operando2: Variables o literales numéricos
  • operador: Uno de los operadores aritméticos soportados

Ejemplos

// Operaciones con variables
t1 = a + b
t2 = t1 * 4
t3 = t2 - 10

// Operaciones con literales
x = 5 + 3
y = x * 2
z = y / 4

// Operaciones mixtas
resultado = x + 100
temp = resultado * y

Evaluación

Las expresiones se evalúan de derecha a izquierda. Primero se resuelven los operandos (obteniendo sus valores), luego se aplica el operador, y finalmente se almacena el resultado en la variable destino.

Nota sobre división por cero: La división por cero genera un error de ejecución que detiene el programa.

4. Expresiones Relacionales y Lógicas

Los operadores relacionales permiten comparar valores. El resultado de una comparación es un valor booleano representado numéricamente: 1 para verdadero y 0 para falso.

Operadores Relacionales

Operador Descripción Ejemplo Resultado
> Mayor que t1 = a > b 1 si a > b, sino 0
< Menor que t2 = a < b 1 si a < b, sino 0
>= Mayor o igual t3 = a >= b 1 si a >= b, sino 0
<= Menor o igual t4 = a <= b 1 si a <= b, sino 0
== Igual a t5 = a == b 1 si a == b, sino 0
!= Diferente de t6 = a != b 1 si a != b, sino 0

Operadores Lógicos

Operador Descripción Ejemplo
&& AND lógico t1 = a && b
|| OR lógico t2 = a || b

Ejemplos

// Comparaciones simples
t1 = a > b
t2 = x <= 10
t3 = contador == 0

// Uso en decisiones
flag = i < n
if flag > 0 goto Loop

// Operadores lógicos
t4 = a > 0 && b > 0
t5 = x < 0 || y < 0

Semántica de Evaluación

Los operadores relacionales y lógicos se evalúan según las reglas estándar:

  • Se obtienen los valores numéricos de ambos operandos
  • Se realiza la comparación o operación lógica
  • El resultado es 1 (verdadero) o 0 (falso)
  • Este resultado se almacena en la variable destino

5. Labels (Etiquetas)

Las etiquetas son marcadores que identifican puntos específicos en el código. Son fundamentales para el control de flujo, ya que sirven como destinos para las instrucciones de salto.

Sintaxis

NombreEtiqueta:

Una etiqueta consiste en un identificador seguido de dos puntos (:). Debe aparecer en su propia línea.

Características

  • Definen bloques básicos: Una etiqueta marca el inicio de un nuevo bloque básico
  • Destinos de salto: Son referenciadas por instrucciones goto e if ... goto
  • Únicas: Cada etiqueta debe ser única en el programa
  • Resolución: Las etiquetas se resuelven antes de la ejecución del programa

Ejemplos

// Etiqueta simple
Inicio:
    x = 0

// Etiqueta para bucle
Loop:
    x = x + 1
    if x < 10 goto Loop

// Etiqueta para condición
Mayor:
    print 1
    goto Fin
    
Menor:
    print 0
    
Fin:
    end

Bloques Básicos

Un bloque básico es una secuencia de instrucciones donde:

  • El flujo de control entra únicamente por el principio (la etiqueta)
  • El flujo de control sale únicamente por el final
  • No hay saltos en medio del bloque

Las etiquetas marcan el inicio de cada bloque básico. Un bloque termina cuando:

  • Se encuentra una instrucción goto o if ... goto
  • Se encuentra una instrucción end
  • Se encuentra otra etiqueta

6. Goto Incondicional

La instrucción goto realiza una transferencia incondicional del control de flujo hacia la etiqueta especificada.

Sintaxis

goto Etiqueta

Comportamiento

Cuando se ejecuta una instrucción goto:

  1. Se busca la etiqueta especificada en el programa
  2. La ejecución continúa inmediatamente después de dicha etiqueta
  3. No se evalúa ninguna condición
  4. El salto siempre ocurre

Ejemplo: Bucle Infinito

Loop:
    print 1
    goto Loop

Este código imprime 1 indefinidamente hasta alcanzar el límite de iteraciones.

Ejemplo: Saltar Código

x = 10
goto Fin

// Este código nunca se ejecuta
x = 20
print x

Fin:
print x
end

Resultado: Imprime 10 (el código intermedio se salta).

Uso en Compiladores

En el contexto de código intermedio, el goto incondicional se utiliza para:

  • Implementar bucles
  • Salir de estructuras de control
  • Simplificar el flujo de control eliminando estructuras anidadas

Nota: Si la etiqueta no existe, se genera un error de ejecución.

7. Goto Condicional

La instrucción if ... goto realiza una transferencia condicional del control de flujo. El salto solo ocurre si la condición evaluada es verdadera.

Sintaxis

if condición goto Etiqueta

Donde condición es una expresión relacional que se evalúa a verdadero o falso.

Comportamiento

Cuando se ejecuta una instrucción if ... goto:

  1. Se evalúa la condición
  2. Si la condición es verdadera (valor diferente de 0):
    • Se realiza el salto a la etiqueta especificada
  3. Si la condición es falsa (valor 0):
    • La ejecución continúa secuencialmente con la siguiente instrucción

Ejemplo: Bucle While

// Simula: while (i < 10) { ... }
i = 0

Loop:
if i >= 10 goto End
print i
i = i + 1
goto Loop

End:
end

Imprime los números del 0 al 9.

Ejemplo: Condicional If-Else

// Simula: if (a > b) { ... } else { ... }
t1 = a > b
if t1 == 0 goto Else

// Rama then
print 1
goto Fin

Else:
// Rama else
print 0

Fin:
end

Operadores en Condiciones

Las condiciones pueden usar cualquier operador relacional o lógico soportado:

  • if x > 0 goto Positivo
  • if a <= b goto Menor
  • if flag == 1 goto Activo
  • if x != 0 goto Continuar

Representación en el Grafo

En el grafo de flujo de control, una instrucción if ... goto genera:

  • Un edge etiquetado "true" hacia el bloque destino (si la condición es verdadera)
  • Un edge etiquetado "false" hacia el siguiente bloque secuencial (si la condición es falsa)

8. Funciones Especiales

El lenguaje proporciona dos funciones especiales para control de ejecución y salida.

Instrucción print

Sintaxis

print expresión

La instrucción print envía un valor al panel de salida en la interfaz del playground.

Comportamiento

  • Evalúa la expresión (variable o literal)
  • Convierte el resultado a cadena de texto
  • Muestra el valor en el panel de salida
  • La ejecución continúa con la siguiente instrucción

Ejemplos

// Imprimir literal
print 42

// Imprimir variable
x = 100
print x

// Imprimir resultado de expresión
a = 5
b = 3
c = a + b
print c

Salida: 42, 100, 8

Instrucción end

Sintaxis

end

La instrucción end detiene la ejecución del programa inmediatamente.

Comportamiento

  • Termina la ejecución del intérprete
  • No se ejecutan más instrucciones después de end
  • Es obligatoria para indicar el fin del programa

Ejemplo

x = 10
print x
end

// Este código nunca se ejecuta
x = 20
print x

Salida: Solo 10. La segunda parte del código se ignora.

Uso en Control de Flujo

La instrucción end se suele colocar al final del flujo principal, después de que todas las ramas de ejecución converjan.

Ejemplo: Estructura Completa

n = 5
if n > 0 goto Positivo

print 0
goto Fin

Positivo:
print 1

Fin:
end

9. Modelo de Ejecución

El intérprete de C3D implementa un modelo de ejecución secuencial con soporte para saltos y control de flujo.

Componentes del Intérprete

1. Tabla de Variables

El intérprete mantiene una tabla de símbolos (implementada como un Map) que asocia cada identificador de variable con su valor actual.

  • Las variables se crean dinámicamente al asignarles un valor
  • Todos los valores son numéricos (punto flotante)
  • El alcance es global (todas las variables están disponibles en todo el programa)

2. Tabla de Etiquetas

Durante la fase de análisis, el parser construye una tabla de etiquetas que mapea cada nombre de etiqueta a su posición (índice de instrucción) en el programa.

  • Se resuelve antes de la ejecución
  • Permite saltos eficientes sin búsqueda lineal
  • Valida que todas las etiquetas referenciadas existan

3. Program Counter (PC)

El contador de programa mantiene el índice de la instrucción actual en ejecución.

  • Inicia en 0 (primera instrucción)
  • Se incrementa secuencialmente después de cada instrucción
  • Se modifica directamente por las instrucciones de salto (goto, if ... goto)

Proceso de Ejecución

La ejecución de un programa C3D sigue estos pasos:

  1. Fase de Análisis (Parse):
    • El código fuente se divide en líneas
    • Se identifican y validan las etiquetas
    • Cada línea se parsea en una estructura de instrucción
    • Se construye la tabla de etiquetas
  2. Fase de Ejecución:
    • PC = 0
    • Mientras PC < número de instrucciones:
      • Leer instrucción en PC
      • Ejecutar instrucción:
        • Resolver operandos (variables o literales)
        • Aplicar operación
        • Actualizar variables o realizar salto
      • Si es end, terminar
      • PC = PC + 1 (o valor del salto)

Manejo de Errores

El intérprete detecta y reporta los siguientes errores:

  • Variable no definida: Se intenta usar una variable antes de asignarle un valor
  • Etiqueta no encontrada: Un goto referencia una etiqueta inexistente
  • División por cero: Se intenta dividir entre 0
  • Sintaxis inválida: Una línea no coincide con ninguna forma válida
  • Bucle infinito: Se alcanza el límite de 100,000 iteraciones

Optimizaciones

Esta implementación prioriza claridad y corrección sobre rendimiento. No se aplican optimizaciones como:

  • Eliminación de código muerto
  • Propagación de constantes
  • Plegado de constantes
  • Eliminación de subexpresiones comunes

10. Representación Gráfica (GUI)

El playground genera automáticamente una representación visual del grafo de flujo de control del programa usando Mermaid.js.

Generación de Bloques Básicos

El proceso de construcción del grafo consta de dos fases:

Fase 1: Identificación de Bloques

Un bloque básico se crea cuando:

  • Se encuentra una nueva etiqueta
  • Se ejecuta una instrucción de control de flujo (goto, if ... goto)
  • Se alcanza el final del programa

Cada bloque contiene:

  • Nombre de la etiqueta que lo identifica
  • Lista de instrucciones que contiene
  • Índice de inicio y fin en el programa

Fase 2: Construcción de Aristas

Las aristas (edges) representan el flujo de control entre bloques. Se crean según estas reglas:

  • Si el bloque termina con goto Label: se crea un edge hacia el bloque Label
  • Si el bloque termina con if ... goto Label: se crean dos edges:
    • Uno hacia Label (etiquetado "true")
    • Otro hacia el siguiente bloque secuencial (etiquetado "false")
  • Si el bloque no termina con salto: se crea un edge hacia el siguiente bloque secuencial
  • Si el bloque termina con end: no se crean edges (es un nodo terminal)

Visualización con Mermaid

El grafo se renderiza usando la sintaxis graph TD (Top-Down) de Mermaid.

Nodos

Cada bloque básico se representa como un nodo rectangular que contiene:

  • El nombre de la etiqueta (encabezado)
  • Las primeras instrucciones del bloque (hasta 4)
  • Indicador "..." si hay más instrucciones

Ejemplo de Nodo

Loop["Loop
if i > n goto End
result = result * i
i = i + 1
goto Loop"]

Aristas (Edges)

Tipo de Edge Representación Descripción
Secuencial Bloque1 --> Bloque2 Flujo secuencial normal entre bloques consecutivos
Goto incondicional Bloque1 --> Target Salto directo sin evaluación de condición
If-goto (true) Bloque1 -->|true| Target Salto condicional cuando la condición es verdadera
If-goto (false) Bloque1 -->|false| Siguiente Flujo secuencial cuando la condición es falsa

Ejemplo Completo

Código C3D

n = 5
result = 1
i = 1

Loop:
if i > n goto End
result = result * i
i = i + 1
goto Loop

End:
print result
end

Bloques Básicos Generados

  • INICIO: n = 5, result = 1, i = 1
  • Loop: if i > n goto End, result = result * i, i = i + 1, goto Loop
  • End: print result, end

Aristas Generadas

  • INICIO → Loop (secuencial)
  • Loop → End (condicional true: i > n)
  • Loop → Loop (condicional false + goto explícito)

Interpretación del Grafo

El grafo de flujo de control permite:

  • Análisis de alcance: Identificar qué bloques son alcanzables desde el inicio
  • Detección de bucles: Visualizar ciclos en el flujo de control
  • Complejidad ciclomática: Evaluar la complejidad del programa
  • Código muerto: Identificar bloques que nunca se ejecutan
  • Optimización: Base para aplicar transformaciones de código
